🎯 Simulasi Rekursi: sum(n)
Hasil Akhir: —
✨ Solusi Kode Rekursif
function sum(n) {
    if (n == 1) return 1; // 🔹 BASE CASE (halaman 6-7)
    else return n + sum(n-1); // 🔁 RECURSIVE CASE
}

Base case: n == 1 → berhenti. Recursive case: n + sum(n-1)

📚 Call Stack (LIFO) — Halaman 8-9
FASE MASUK (PUSH) — Suspended Calls
⏳ Belum ada eksekusi
FASE MUNDUR (POP) — Unwinding / Menggabungkan
Tekan "Eksekusi Rekursi"

📖 Penjelasan Berdasarkan Materi "Struktur Data pert. 7.pdf"

🎯 Base Case (Kasus Dasar) – hlm. 6-7
Fungsi rekursif WAJIB memiliki kondisi berhenti. Pada challenge ini: if (n == 1) return 1;
➜ Tanpa base case, terjadi infinite recursion → Stack Overflow (hlm. 10).
🔄 Recursive Case (Kasus Rekursif) – hlm. 5-6
Fungsi memanggil dirinya sendiri dengan skala lebih kecil: return n + sum(n-1);
Analogi "Forward Message berantai" di grup chat (hlm. 5).
🧠 Call Stack & LIFO – hlm. 8-9 (Menyelam & Mundur)
Fase Masuk (PUSH): Setiap pemanggilan fungsi ditumpuk di memori. Eksekusi ditunda hingga base case tercapai.
Fase Mundur (POP): Setelah base case mengembalikan nilai, stack dibongkar dari atas ke bawah, hasil digabungkan seperti efek domino mundur.
➜ Prinsip Last In, First Out persis seperti contoh faktorial f(3) di materi.
⚠️ Sisi Gelap: Stack Overflow – hlm. 10
Jika base case tidak tepat atau rekursi tak terbatas, memori akan habis dan program crash.
Rekursi vs Iterasi (hlm.12): Rekursi lebih elegan untuk nested data (tree/folder), namun boros memori dibanding loop biasa.
🌍 Implementasi Dunia Nyata – hlm. 13-14
Sistem file/folder (cari file di dalam folder), AI & pohon keputusan (Minimax), pathfinding. Rekursi unggul untuk struktur bercabang.
💡 Kesimpulan Emas (hlm.16)
"If a problem looks like itself, recursion is your best friend." — Gunakan rekursi dengan hati-hati, pastikan base case & pahami LIFO mindset.
